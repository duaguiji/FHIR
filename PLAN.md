系統規劃文件：跨場域運動生理監測與自動化評估系統
1. 情境敘述 (Context)
民眾在校園健身房（如台科大健身房）或公共運動中心運動後，身體通常處於高度代謝與負荷狀態。為了確保運動安全並追蹤體能表現，民眾會到場邊的檢測站，使用血氧儀或心率感測器量測生理數值。目前這些數值通常只能在設備上短暫顯示，或是記錄在各個場館不互通的 App 中，導致使用者無法長期且跨場域地整合自己的運動健康資料。

2. 情境目標 (Goals)
解決數據碎片化： 透過國際標準 FHIR 格式，讓不同場館、不同設備量測到的運動數據都能交換並統一儲存。

即時風險預警： 民眾往往不清楚量測到的數字意義，本系統旨在提供「即時評估」，若血氧過低或心率異常，立即給予安全指引。

提升健康意識： 讓使用者能更有感地看到運動後的生理變化，減少運動過度導致的傷害。

3. 需求分析 (Requirement Analysis)
身分識別功能： 使用者能透過簡單的編號（Patient ID）識別身分並與場所（Location）關聯。

數據輸入與顯示： 提供直覺的介面讓使用者輸入心率與血氧值，並能即時看到數值結果。

自動化評估建議： 系統需內建生理邏輯，根據輸入數值自動判定健康等級（正常/警示）並顯示文字建議。

標準化資料產出： 系統必須能將零散數據封裝成符合 FHIR 標準的 Observation JSON 格式。

4. 工作流程 (Workflow)
使用者登入與情境選擇： 民眾抵達檢測站，掃描場館 QR Code（或手動選擇場所），系統讀取使用者基本資料 (Patient) 與場所資訊 (Location)。

生理數值量測： 使用者使用感測設備測量，並將「心率」與「血氧」數值填入網頁系統。

FHIR 轉換與評估： 系統接收數據後：

將數值轉換成符合國際編碼（如 LOINC）的 Observation 資源格式。

同步啟動邏輯判定：比對血氧是否低於 95% 或心率是否過高。

建議反饋： 網頁即時顯示評估結果與建議（例如：顯示「血氧偏低，請坐下休息」）。

雲端存儲： 使用者確認無誤後，系統將資料傳送至雲端 FHIR Server 進行存儲，供未來長期分析。

5. 使用角色 (User Roles)
一般民眾（運動者）： 系統的核心使用者，負責輸入量測數值並接收運動安全建議。

場域管理員（如教練或救生員）： 透過管理後台監測場內使用者的異常生理警示，在必要時提供協助。

醫療/健康管理師： 透過系統累積的長期 FHIR 數據，評估使用者的體能進步狀況並給予專業指導。

6. 主要 FHIR Resource (Core Resources)
Patient: 記錄運動者的基本資料（如學號、性別、年齡）。

Observation: 最核心的資料格式，用於記錄心率 (Heart Rate) 與血氧 (SpO2) 數值及產生的評估建議。

Location: 記錄運動發生的場所資訊，如「台科大健身房」。

Organization: 記錄場館的隸屬單位，如「台科大體育室」。
