# 系統規劃文件：專業級運動生理監測系統 (APSH)

## 1. 情境目標 (Goals)
* **解決數據碎片化**：透過 FHIR Patient 與 Observation 的連結，建立完整的個人運動健康紀錄 (PHR)。
* **即時處置引導**：不只提供警告，更針對異常時間點給予「坐下休息」、「停止運動」或「立即就醫」等行為指引。
* **資料正確性維護**：透過前端欄位限制與刪除機制，防止錯誤數據影響分析趨勢。

## 2. 核心技術架構
* **前端技術**：HTML5, CSS3, JavaScript (Vanilla JS), Chart.js。
* **API 標準**：HL7 FHIR R4 規範。
* **診斷引擎邏輯**：
    * 系統接收數據後，自動與內建的 **ISH/AHA 臨床標準** 進行比對。
    * **多維度掃描**：同步判斷「血壓分級」與「高溫環境熱衰竭風險」。

## 3. 異常監控邏輯 (Health Logic)
* **時間定位分析**：當系統偵測到 Level ≥ 2 的異常時，會擷取該筆 Observation 的 `effectiveDateTime`。
* **診斷回饋串接**：將診斷結果 (`getDiagnosis`) 與數據列表整合，在 UI 頂部生成彙整型警告區，讓使用者能一眼看出今日的問題發生點。

## 4. 使用者流程 (Workflow)
1. **身分建立/驗證**：註冊新 Patient 或輸入 ID 讀取雲端資料。
2. **數據錄入**：輸入生理指標，系統自動補抓當下時間（不允許未來日期）。
3. **雲端同步**：透過 POST 將數據與使用者 ID 綁定並存儲。
4. **回溯分析**：使用者隨時可調閱單日詳細明細或週趨勢圖，並能手動修正錯誤數據。
