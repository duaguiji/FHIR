# 系統規劃文件：跨場域運動生理監測與自動化評估系統 (APSH)

## 1. 情境目標 (Goals)
* **數據整合透明化**：解決運動數據散落在不同設備或 App 的問題。
* **智慧化診斷 (CDSS)**：讓冷冰冰的數字變成有意義的「處置建議」。
* **精準訓練管理**：透過週平均值的變動，協助運動員或一般民眾判斷是否「過度訓練」。

## 2. 核心技術架構
* **前端技術**：HTML5, CSS3, JavaScript (Vanilla JS), Chart.js (數據視覺化)。
* **後端標準**：HL7 FHIR R4 規範。
* **數據聚合算法**：
    * 系統自動將同一天內的多筆 Observation 數據，依據 `effectiveDateTime` 歸類。
    * 在週模式下，計算 `Σ(value) / n` 作為該日的代表數據點。

## 3. 異常監控邏輯 (Health Logic)
* **單日診斷**：針對每筆資料的即時判定。
* **週全域掃描**：即使當天平均值正常，系統仍會掃描原始數據。若發現任何一次量測達到「危急值 (SBP ≥ 180)」，將在週報表頂端彈出紅色警告，確保不漏掉任何潛在風險。

## 4. 使用者流程 (Workflow)
1. **數據採集**：使用者輸入生理指標，系統將其封裝為多 component 的 Observation。
2. **雲端同步**：透過 RESTful API (POST) 將 JSON 送往伺服器。
3. **動態查詢**：使用者不需上傳即可隨時輸入 ID 進行回溯查詢。
4. **多模式分析**：使用者根據需求切換「詳細清單」或「平均趨勢圖」。
