# AnyPlace Sport-Health (APSH) - 專業級運動生理監測與自動化評估系統

## 1. 專案簡介
本專案針對民眾在不同運動場所運動後，缺乏專業生理評估與數據追蹤的痛點。系統支援使用者即時登記運動後的「血壓（SBP/DBP）」、「心率（HR）」及「環境氣溫」，並將數據轉換為 **FHIR 國際標準格式**。

系統核心特色在於具備 **CDSS (臨床決策支持系統)** 雛形，能針對「單日多筆紀錄」進行全量掃描，精確定位特定時間點的異常並給予行動指引。

## 2. 核心 FHIR Resources
* **Patient**: 實作使用者註冊與登入驗證，確保生理紀錄與個人身分永久關聯。
* **Observation**: 採用多組件 (Component) 資源，將血壓、心率、環境氣溫與運動類別封裝於單次量測紀錄；環境氣溫代表運動場域氣溫，不作為體溫或發燒判斷。

## 3. 系統功能亮點
* **完整使用者管理 (CRUD)**：
    * 提供新用戶註冊（Create Patient）與舊用戶 ID 驗證登入（Read Patient）。
    * 資料持久化：自動記憶登入狀態，提升使用體驗。
* **智慧化診斷與精準定位**：
    * **單日列表模式**：自動掃描當天所有紀錄，若有多筆量測，系統會精確標註「幾點幾分」出現異常，並提供分級建議。
    * **週趨勢分析**：自動對計算週一至週日平均值，並同步進行「異常全量掃描」。
* **數據修正功能**：支援單筆 Observation 刪除 (Delete)，確保歷史數據的真實性。
* **數據邏輯防錯**：自動限制出生日期與量測時間不可選擇未來日期。
* **AI 智慧診斷**：整合 Google Gemini API，提供專業的運動生理分析和健康建議。
* **現代化介面設計**：採用響應式佈局、專業色彩方案與流暢動畫，提升用戶操作體驗。

## 4. 如何執行 (How to Run)
1. 使用 VS Code 開啟資料夾，確保 `index.html` 與 `script.js` 處於同目錄。
2. 使用 **Live Server** 或本地 HTTP 服務器開啟 `index.html`。
3. 進行新用戶註冊後，即可開始上傳與分析數據。

## 5. AI 診斷功能設置 (Gemini API)
系統整合了 Google Gemini AI 提供智慧診斷分析：

### 設置步驟
1. 前往 [Google AI Studio](https://makersuite.google.com/app/apikey) 獲取 API Key
2. 在 `script.js` 中將 `GEMINI_API_KEY` 設置為您的實際 API Key
3. 重新載入網頁即可使用 AI 診斷功能

### 功能特色
- **專業分析**：基於運動生理學知識提供健康評估
- **個性化建議**：根據您的具體數據給出針對性建議
- **風險預警**：識別潛在健康風險並提供預防建議

詳細設置說明請參考 `GEMINI_SETUP.md` 文件。

## 6. 介面特色
- **簡潔導航**：頂部導航欄提供快速頁面切換
- **響應式設計**：支援桌面與移動設備
- **專業視覺**：使用現代 CSS 變數、漸層背景與陰影效果
- **用戶友好**：清晰的表單標籤、按鈕狀態與視覺回饋
