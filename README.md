# AnyPlace Sport-Health (APSH) - 跨場域運動生理監測與自動化評估系統

## 1. 專案簡介
本專案針對民眾在不同運動場所（如學校健身房、國民運動中心等）運動後，缺乏生理數據長期追蹤與專業健康評估的痛點。本系統支援使用者即時登記運動後的「血壓（SBP/DBP）」、「心率（HR）」、「血氧」及「環境氣溫」，並將數據轉換為 **FHIR 國際標準格式**。

系統核心特色在於其 **「生理數據回溯分析儀表板」**，能自動計算週一至週日的日平均趨勢，並根據 **國際高血壓學會 (ISH)** 標準，提供五個等級的自動化診斷與行動處置建議。

## 2. 競賽必要資訊
* **主題領域**： 運動健康 / 預防醫學 / 智慧醫療
* **使用者角色**： 
    * **運動民眾**：回溯查詢生理數據、接收針對性的醫療級運動建議。
    * **場域管理員/教練**：監測場內使用者的長期疲勞狀況與即時異常風險。
* **核心 FHIR Resources**：
    * **Patient**: 使用者身分識別（學號、個人資料）。
    * **Observation**: 封裝血壓、心率、氣溫及運動類別的多組件（Component）資源。
    * **Location**: 標記量測場所（如：台科大體育館）。

## 3. 系統功能亮點
* **雙模式分析切換**：
    * **單日模式**：以「時間軸卡片」呈現，提供每筆紀錄的即時診斷。
    * **週模式**：自動以「週一至週日」為 X 軸，計算日平均值，觀察長期體能恢復趨勢。
* **五級血壓臨床警示**：系統自動識別「正常、偏高、一級高血壓、二級高血壓、危急值」，並給予具體行動指引。
* **三軌獨立視覺化**：血壓、心率、氣溫獨立分圖，避免數值單位干擾，提升數據易讀性。

## 4. 如何執行 (How to Run)
1. 使用 VS Code 開啟資料夾。
2. 確保 `index.html` 與 `script.js` 處於同目錄。
3. 使用 **Live Server** 開啟 `index.html`。
4. 於查詢頁面輸入 Patient ID (如: 110) 與日期，即可直接調取雲端數據進行分析。
